1. Important Terms You Need to Know

Before we start, let's learn some vocabulary:

Term What It Means (Simple Explanation)
Repository (Repo) A folder on the internet that holds all your project files. Think of it like a Google Drive folder, but specifically for code.
GitHub A website where programmers store their code. It's like Instagram, but instead of photos, people share code.
Hosting Putting your website on the internet so anyone can see it.
Deploy The action of putting your website online. Like hitting "publish" on a blog post.
Environment Variable (env) A secret value (like a password) that your website needs but you don't want anyone else to see.
API Key A special password that lets your website talk to another service (like Pollination.ai).
Terminal/Command Line A text-based way to control your computer by typing commands instead of clicking.
Clone Making a copy of a repository to your computer.
Commit Saving your changes with a description of what you changed.
Push Sending your saved changes from your computer to GitHub.
Branch A separate version of your project. The main one is usually called "main" or "master."

2. GitHub Pages Setup

What is GitHub Pages?

GitHub Pages is a FREE way to put a simple website on the internet. It's perfect for basic websites but has limitations (we'll discuss these later).

Step 1: Create a GitHub Account

  1. Open your web browser (Chrome, Firefox, Safari, Edge - whatever you use)
  2. Go to https://github.com
  3. Click the "Sign Up" button in the top-right corner
  4. Enter your email address and click "Continue"
  5. Create a password (make it strong - at least 8 characters with numbers and letters)
  6. Choose a username (this will be part of your website address, so pick something appropriate)
  7. Complete the puzzle to prove you're human
  8. Click "Create Account"
  9. Check your email for a verification code
  10. Enter the code on the GitHub website
  11. You now have a GitHub account!

Step 2: Create Your First Repository

  1. Make sure you're logged into GitHub
  2. Look at the top-right corner of the screen - you should see a "+" symbol
  3. Click the "+" symbol
  4. Click "New repository" from the dropdown menu

You're now on the "Create a new repository" page.

  1. In the "Repository name" box, type a name for your project
    • Example: my-first-website
    • No spaces allowed! Use dashes (-) instead
  2. In the "Description" box (optional), write what your project is about
    • Example: "My first website using GitHub Pages"
  3. Select "Public"
    • This means anyone can see your code
    • GitHub Pages only works for free with public repositories
  4. Check the box that says "Add a README file"
    • A README is like a welcome note explaining your project
  5. Click the green "Create repository" button at the bottom
üéâ Congratulations! You just created your first repository!

Step 3: Create Your Website Files

Now we need to add actual website files to your repository.

Creating an HTML file (your main webpage):

  1. You should be on your repository page (it shows your repository name at the top)
  2. Click the "Add file" button (it's a dropdown near the top of the page)
  3. Click "Create new file"
  4. In the "Name your file..." box, type: index.html
    • This MUST be named exactly index.html - this is your homepage
  5. In the big text box below, copy and paste this code:
<!DOCTYPE html>

            padding: 20px;

        }

            color: #333;
        }
        .container {
            background-color: white;
            padding: 20px;
            border-radius: 10px;
            box-shadow: 0 2px 5px rgba(0,0,0,0.1);
        }
    </style>

<body>

        <h1>Welcome to My Website!</h1>
        <p>This is my first website hosted on GitHub Pages.</p>
        <p>I built this by following a tutorial!</p>
    </div>

  1. Scroll down to the "Commit new file" section
  2. In the first text box, type a message describing what you did
    • Example: "Add homepage"
  3. Make sure "Commit directly to the main branch" is selected
  4. Click the green "Commit new file" button

Step 4: Turn on GitHub Pages

  1. Click on "Settings" - it's a tab near the top of your repository page (looks like a gear icon)
  2. Look at the left sidebar (the menu on the left side)
  3. Scroll down and click "Pages"
  4. Under "Source", you'll see a dropdown that says "None"
  5. Click the dropdown and select "main" (or "master" if that's what it shows)
  6. Click "Save"
  7. Wait about 1-2 minutes
  8. Refresh the page (press F5 or click the refresh button)
  9. You should see a message that says "Your site is live at..."
  10. Click that link!
üéâ Your website is now live on the internet!

Your website address will be: https://YOUR-USERNAME.github.io/YOUR-REPOSITORY-NAME/

3. Vercel Setup

What is Vercel?

Vercel is a company that hosts websites for FREE. It's more powerful than GitHub Pages and can handle more complex websites. It also makes it easier to use secret API keys.

Step 1: Create a Vercel Account

  1. Go to https://vercel.com
  2. Click "Sign Up" in the top-right corner
  3. Click "Continue with GitHub" (this is the easiest way)
  4. A popup will appear asking you to authorize Vercel
  5. Click "Authorize Vercel"
  6. You might need to enter your GitHub password
  7. You now have a Vercel account connected to GitHub!

Step 2: Import Your GitHub Repository to Vercel

  1. After signing up, you should be on the Vercel dashboard
  2. Click "Add New..." button
  3. Click "Project"
  4. You'll see a list of your GitHub repositories
  5. Find the repository you created earlier (like my-first-website)
  6. Click "Import" next to it
If you don't see your repository:
  • Click "Adjust GitHub App Permissions"
  • Select your account
  • Under "Repository access," select "All repositories" or specifically add your repository
  • Click "Save"
  • Go back to Vercel and refresh

Step 3: Configure Your Project

  1. You're now on the "Configure Project" page
  2. Project Name: You can keep the default or change it
    • This will be part of your URL
  3. Framework Preset: Leave as "Other" for a basic HTML website
  4. Root Directory: Leave empty (or "./")
  5. For now, skip the "Environment Variables" section (we'll come back to this)
  6. Click "Deploy"

Step 4: Wait for Deployment

  1. Vercel will now build and deploy your website
  2. You'll see a progress screen with logs (text showing what's happening)
  3. Wait about 30 seconds to 1 minute
  4. When it's done, you'll see "Congratulations!" and a preview of your website
  5. Click "Continue to Dashboard"

Step 5: Find Your Website URL

  1. On your project dashboard, look for "Domains"
  2. You'll see a URL like your-project-name.vercel.app
  3. Click it to see your live website!
üéâ Your website is now also live on Vercel!

4. Netlify Setup

What is Netlify?

Netlify is another FREE website hosting service, similar to Vercel. Some people prefer it, and it's good to know both!

Step 1: Create a Netlify Account

  1. Go to https://www.netlify.com
  2. Click "Sign up" in the top-right corner
  3. Click "GitHub" to sign up with your GitHub account
  4. Click "Authorize Netlify" when the popup appears
  5. You might need to enter your GitHub password
  6. You now have a Netlify account!

Step 2: Create a New Site from GitHub

  1. You should be on the Netlify dashboard (called "Team overview")
  2. Click "Add new site" button
  3. Click "Import an existing project"
  4. Click "Deploy with GitHub"
  5. A popup appears - click "Authorize Netlify"
  6. Select your repository from the list
If you don't see your repository:
  • Click "Configure Netlify on GitHub"
  • Select your account
  • Choose "All repositories" or select specific ones
  • Click "Save"
  • Go back and refresh

Step 3: Configure Your Site

  1. Branch to deploy: Should say "main" (leave it)
  2. Base directory: Leave empty
  3. Build command: Leave empty (for basic HTML sites)
  4. Publish directory: Leave empty or type .
  5. Click "Deploy site"

Step 4: Wait for Deployment

  1. Netlify will build your site
  2. This takes about 30 seconds to 2 minutes
  3. When done, you'll see "Published" with a green checkmark

Step 5: Find Your Website URL

  1. On your site dashboard, look at the top
  2. You'll see a URL like random-name-123456.netlify.app
  3. Click it to see your live website!

Optional: Change Your Site Name

  1. Click "Site configuration" (or "Site settings")
  2. Click "Change site name"
  3. Type a new name (like my-cool-website)
  4. Click "Save"
  5. Your new URL will be my-cool-website.netlify.app
üéâ Your website is now also live on Netlify!

5. Setting Up Your Pollination.ai API Key

‚ö†Ô∏è VERY IMPORTANT WARNING ‚ö†Ô∏è

API keys are like passwords. If someone gets your API key, they can:

  • Use your account
  • Possibly charge money to you
  • Do bad things pretending to be you

NEVER put your API key directly in your code files!

NEVER share your API key with anyone!

NEVER upload files containing your API key to GitHub!

Understanding the Problem with GitHub Pages

Here's something important: GitHub Pages cannot keep secrets safe.

When you use GitHub Pages:

  • All your files are public
  • Anyone can see your code
  • If you put an API key in your code, anyone can steal it
Solution: For projects that need API keys, use Vercel or Netlify instead of GitHub Pages. They have special secure storage for secrets.

Getting Your Pollination.ai API Key

  1. Go to https://enter.pollination.ai
  2. Create an account by logging in via your GitHub account
  3. Look for "API Keys,"
  4. Generate a new API key
  5. Choose the models you want you key to work with (you can create multiple keys)
  6. Copy the key and save it somewhere safe (like a password manager)
  7. NEVER share this with anyone!
  8. Note: Pollinations Public Keys can be copied later, Secret Keys will only let you copy them one time when they are first generated.

Setting Up Environment Variables in Vercel

What is an Environment Variable?

It's a secret value that Vercel knows about, but nobody else can see. Your website code can use it, but it's not visible to visitors.

Step-by-Step Instructions:

  1. Go to https://vercel.com
  2. Log in if you're not already
  3. Click on your project name to open it
  4. Click "Settings" (tab at the top)
  5. Click "Environment Variables" (in the left sidebar)

Now you'll add your API key:

  1. In the "Key" box, type a name for your variable
    • Example: POLLINATION_API_KEY
    • Use ALL CAPS and underscores, no spaces
  2. In the "Value" box, paste your actual API key
    • Example: sk-abc123xyz789... (your real key)
  3. Under "Environment", make sure all three are checked:
    • ‚úÖ Production
    • ‚úÖ Preview
    • ‚úÖ Development
  4. Click "Save"

Creating a Serverless Function in Vercel

Basic HTML/JavaScript sites cannot directly access environment variables because the code runs in the visitor's browser. You need to use a "serverless function" - a small piece of code that runs on Vercel's servers.

  1. Go to your repository on GitHub
  2. Click "Add file" ‚Üí "Create new file"
  3. Type the name: api/pollination.js
    • The api/ part creates a folder called "api"
    • This tells Vercel "this is a serverless function"
  4. Paste this code:
// This is a serverless function
// It runs on Vercel's servers, not in the browser
// So it can safely access your secret API key

export default async function handler(request, response) {
    // Get the secret API key from environment variables
    const apiKey = process.env.POLLINATION_API_KEY;
    
    // Check if the API key exists
    if (!apiKey) {
        return response.status(500).json({ 
            error: 'API key not configured' 
        });
    }
    
    try {
        // Make a request to Pollination.ai
        // You'll need to adjust this URL based on Pollination's documentation
        const pollinationResponse = await fetch('https://api.pollination.ai/your-endpoint', {
            method: 'POST',
            headers: {
                'Authorization': `Bearer ${apiKey}`,
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(request.body)
        });
        
        const data = await pollinationResponse.json();
        
        // Send the response back to your website
        return response.status(200).json(data);
        
    } catch (error) {
        return response.status(500).json({ 
            error: 'Something went wrong' 
        });
    }
}
  1. Commit the file (type a message and click "Commit new file")

Creating a Serverless Function in Netlify

Netlify calls serverless functions just "Functions." Here's how to create one:

  1. Go to your repository on GitHub
  2. Create a new file called netlify.toml in the root folder
  3. Add this content:
[build]
  functions = "netlify/functions"

[functions]
  node_bundler = "esbuild"
  1. Commit the file
  2. Create another new file: netlify/functions/pollination.js
    • This creates a folder called "netlify" with a subfolder called "functions"
  3. Paste this code:
// Netlify serverless function for Pollination.ai

exports.handler = async function(event, context) {
    // Get the secret API key from environment variables
    const apiKey = process.env.POLLINATION_API_KEY;
    
    // Check if the API key exists
    if (!apiKey) {
        return {
            statusCode: 500,
            body: JSON.stringify({ error: 'API key not configured' })
        };
    }
    
    try {
        // Get the data sent from your website
        const requestBody = JSON.parse(event.body || '{}');
        
        // Make a request to Pollination.ai
        const response = await fetch('https://api.pollination.ai/your-endpoint', {
            method: 'POST',
            headers: {
                'Authorization': `Bearer ${apiKey}`,
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(requestBody)
        });
        
        const data = await response.json();
        
        return {
            statusCode: 200,
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(data)
        };
        
    } catch (error) {
        return {
            statusCode: 500,
            body: JSON.stringify({ error: 'Something went wrong: ' + error.message })
        };
    }
};
  1. Commit the file

6. How to Update Your API Key Later

Sometimes you need to change your API key. Maybe it got leaked, or you're using a new account. Here's how to update it on each platform:

Updating on Vercel

  1. Go to https://vercel.com
  2. Log in
  3. Click your project name
  4. Click "Settings" (tab at the top)
  5. Click "Environment Variables" (left sidebar)
  6. Find POLLINATION_API_KEY in the list
  7. Click the three dots (‚ãÆ) on the right side of that row
  8. Click "Edit"
  9. Delete the old value and paste your new API key
  10. Click "Save"
  11. IMPORTANT: You need to redeploy for changes to take effect!
    • Click "Deployments" tab
    • Find the most recent deployment
    • Click the three dots (‚ãÆ)
    • Click "Redeploy"
    • Click "Redeploy" again to confirm

Updating on Netlify

  1. Go to https://app.netlify.com
  2. Log in
  3. Click your site name
  4. Click "Site configuration" (sidebar)
  5. Click "Environment variables"
  6. Find POLLINATION_API_KEY
  7. Click "Options" (three dots) on the right
  8. Click "Edit"
  9. Delete the old value and paste your new API key
  10. Click "Save"
  11. Trigger a redeploy:
    • Click "Deploys" in the sidebar
    • Click "Trigger deploy" button
    • Click "Deploy site"

Deleting an API Key

If you want to completely remove an API key:

On Vercel:

  1. Go to Settings ‚Üí Environment Variables
  2. Find the variable
  3. Click the three dots (‚ãÆ)
  4. Click "Remove"
  5. Confirm deletion

On Netlify:

  1. Go to Site configuration ‚Üí Environment variables
  2. Find the variable
  3. Click "Options"
  4. Click "Delete"
  5. Confirm deletion

8. Troubleshooting Common Problems

"My website shows a 404 error"

  • Make sure your main file is named exactly index.html
  • Make sure GitHub Pages is enabled (Settings ‚Üí Pages)
  • Wait a few minutes - sometimes it takes time to update

"My API key isn't working"

  • Double-check you copied the entire key
  • Make sure there are no extra spaces before or after
  • Verify the variable name matches exactly (including capitals)
  • Redeploy your site after adding/changing the key

"I can't see my repository in Vercel/Netlify"

  • Make sure you've given them permission to access your repositories
  • Try clicking "Configure" or "Adjust GitHub App Permissions"
  • Make sure your repository is public

"My serverless function returns an error"

  • Check the function logs in your dashboard
  • Make sure the file is in the correct folder
  • Verify your API endpoint URL is correct
